# Module 06: System Design

Architect large-scale distributed systems with proper trade-offs, scalability strategies, and reliability patterns.

## Learning Objectives
- Evaluate scalability dimensions and load characteristics
- Apply caching, load balancing, sharding, replication
- Design database schemas & storage selection (SQL/NoSQL)
- Produce architecture diagrams & communication flows
- Reason about CAP theorem & consistency models
- Execute back-of-envelope capacity estimations
- Produce design artifacts: APIs, data flow, component responsibilities

## Submodules
- 01-fundamentals
- 02-patterns
- 03-case-studies
- 04-trade-offs
- 05-estimation

## Estimated Duration
2-3 weeks

## Prerequisites
- Strong Java & networking basics
- Familiarity with microservices patterns
